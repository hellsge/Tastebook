# 提交规范

## 咖啡数据变更

- 当改动涉及咖啡记录数据（如 `coffee.csv` 或页面中的咖啡条目），commit message 直接使用咖啡品名
  - 例：`燕麦拿铁（冰）`
- 如果品名存在重复或无法区分，追加品牌名
  - 例：`拿铁(冰) - 比星`
- 一次提交包含多条咖啡记录时，用顿号分隔
  - 例：`燕麦拿铁（冰）、深烘拿铁(冰)`

## 架构 / 脚本 / 其他改动

使用标准提交格式（Conventional Commits），类型标签使用英文，描述使用中文：

```
<类型>: <中文描述>
```

常用类型：
- `feat`: 新功能
- `fix`: 修复问题
- `docs`: 文档变更
- `style`: 样式调整（不影响逻辑）
- `refactor`: 重构
- `chore`: 构建、配置等杂项

示例：
- `feat: 添加咖啡录入页面`
- `docs: 更新评分标准说明`
- `fix: 修复导出CSV编码问题`

## 通用规则

- 仓库内所有文案、注释、文档尽量使用中文
- commit message 保持简洁，一行以内
